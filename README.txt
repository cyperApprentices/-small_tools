1.留存一些常用的工具和函数，提高代码的复用。

2.不必把精力放在追求高泛用性，适用即可，即插即用,就像一块积木。

3.每一个工具函数，需要一个合适的注释，并注册到README的目录当中。
    注释的例子如下:
        #* NO.2 
        #* 在windows环境下，根据文件的后缀，从文件夹中提取对应后缀的文件
    注释包含：
        1.工具的编号。方便将注册到目录当中。
        2.工具的描述，包括的使用的操作系统，具体的功能

4.每一个文件需要一个使用示例。

5.尽可能的对一些不常用的函数、算法、feature(特性)写清晰注释,slef-documenting(自文档化)
